## 注意！このテンプレートを使用するとプロンプト生成時に処理が走るので一瞬カクつく場合があります！
なんか最近たまにカクつくな？ってなったら十中八九コイツのせいです

## これは何？
RimtalkがLLMに渡す内容を各環境に合わせて取捨選択し、出力精度を上げるためのエントリ群
また、デフォルト値ではLLMに伝わり辛い項目の補足、変更が出来る

「ゴージュースによる興奮状態」→「ゴージュースという戦闘用に開発された薬物を飲み興奮状態です」

「夜のよたか」→「夜に活動するのが好きです」

## どうやって使うの？
Base InstructionとPawn Profiles.txtの内容をそれぞれで上書きし、Pawn ProfilesのロールをUserに変更することで使用可能
JSON Format、Chat History、Dialogue Promptはオフにしてください
コンテキストフィルターのコンテキスト最適化はオフ、含める最大ポーンは2～3、会話履歴セッション数は0～1、ポーン情報と環境は全て許可してください
「ムードと社会的影響を適用」も使用出来ますが、LLMの性能次第では出力精度が落ち、JSONLが崩れたりします

## 導入すると現状何が出来るの？
* 心情、健康、特性情報の取捨選択
* 温度や天気は異常時のみ出力（必要以上に天気の話をしないように）
* 戦闘、狩り時のみ武器情報を出力
* 赤ちゃんが流暢に喋らないよう調整
* 得意スキル（10以上）のものだけを出力（大量のスキル名で混乱させないように）
* 身近な人や絆で結ばれた動物の死をより認識しやすいよう変更
* 室外にいるかどうかのチェック、異常な天気の時のみ室内にいることを強調
* 発言者の派閥情報を明確に（敵と仲良く会話しないように）
* 現在マップが遠征地かどうかを判別
* 現在マップが敵の拠点かどうかを判別
* 会話相手との社交関係のみを出力（他の社交関係を無駄に出力し混乱させないように）
* ポーンのジョブに会話している相手以外の名前が出てきた時にそのポーンの情報（人間かどうか等）を取得、これによって「ブタニクを攻撃中」というジョブがあった時に支離滅裂な文を生成しないように
* 会話人数が何人かを明確にし謎の第三者を出さないように
* 訪問者の種別を登録し、現在コロニーに誰が訪れているか、どんな形式の襲撃が来ているかを把握出来るように
* LLMが絶対に知らないオブジェクト（サイトスティーラー等）の名称が会話やジョブに含まれていた場合、自動で補足説明を挿入出来るように

## 自分のMod盛々環境用に調整したい！
基本的にProfile上部に置いてある辞書に文字列を追加するだけで調整出来るようにしてあります。（▼記号がある箇所）
現状本当に最低限しか設定してないので、この辞書を拡張することで理想の出力が出来るようになると思います
Rimtalkのデバッグ機能は一度LLMにプロンプトを送信した後にプロンプト設定画面のプレビュー画面を押すことで使用可能です
ただし、常にScribanを実行し続ける為かなり重たい上に文字数が多すぎるとスクロール関係の制限で編集しづらいです

## 注意事項
使用モデルはgemmaシリーズレベルを想定しているため、ローカルモデルでは上手く動かない可能性があります
ポーン数は7でも動作確認していますが、台詞の質が下がる上にLLMに渡すプロンプト作成時に確実にカクつきます。（最適化不足+走査）
TPM等にも影響するのでポーン数は2が推奨です

Rimtalkの編集画面でのスクロール限界が来てしまったのでアプデ頻度は下がります、使ってくれている方がいたら要望やバグを教えてもらえたらとても喜びます
