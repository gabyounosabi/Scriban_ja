## これは何？
RimtalkがLLMに渡す内容を各環境に合わせて取捨選択し、出力精度を上げるためのエントリ群
また、デフォルト値ではLLMに伝わり辛い項目の補足、変更が出来る

「ゴージュースによる興奮状態」→「ゴージュースという戦闘用に開発された薬物を飲み興奮状態です」

「夜のよたか」→「夜に活動するのが好きです」

## どうやって使うの？
現状では新規エントリを作成しProfile.txtの内容をそのまま貼り付けし、Base Instruction、Json Formatの下に配置する
その後Dialogue Prompt.txtの内容で既存のDialogue Promptを上書き
ただし今後仕様変更（エントリの分割等）する可能性が高いです

## 導入すると現状何が出来るの？
* 心情、健康、特性情報の取捨選択
* 温度や天気は異常時のみ出力（必要以上に天気の話をしないように）
* 戦闘、狩り時のみ武器情報を出力
* 赤ちゃんが流暢に喋らないよう調整
* 得意スキル（10以上）のものだけを出力（大量のスキル名で混乱させないように）
* 身近な人や絆で結ばれた動物の死をより認識しやすいよう変更
* 室外にいるかどうかのチェック、異常な天気の時のみ室内にいることを強調
* 発言者の派閥情報を明確に（敵と仲良く会話しないように）
* 現在マップが遠征地かどうかを判別
* 現在マップが敵の拠点かどうかを判別
* 会話相手との社交関係を把握（他の社交関係を無駄に出力しないように）
* ポーンのジョブに会話している相手以外の名前が出てきた時にそのポーンの情報（人間かどうか等）を取得、これによって「ブタニクを攻撃中」というジョブがあった時に支離滅裂な文を生成しないように

## 自分のMod盛々環境用に調整したい！
基本的にProfile上部に置いてある辞書に文字列を追加するだけで調整出来るようにしてあります。
現状本当に最低限しか設定してないので、この辞書を拡張することで理想の出力が出来るようになると思います

注意！Rimtalkのエントリ入力欄ではプレビューによりリアルタイムデバッグが出来ますが文字数が多すぎるとスクロール関係の不具合で編集しづらいです

## 注意事項
現在、独り言、奴隷の会話、囚人の会話、他派閥の会話、敵との会話に対応してはいますが、自派閥以外のポーンが選択された場合に特定プロパティが取れず赤エラーを返す事があります（条件不明）
頻繁に起きる場合は他派閥の会話、敵の会話の許可を外してください
使用モデルはgemmaシリーズレベルを想定しているため、ローカルモデルでは上手く動かない可能性があります
思ったように台詞が生成されない場合、開発者モードの詳細ログやプロンプト設定画面のプレビュー等で送信したプロンプトを確認してください
