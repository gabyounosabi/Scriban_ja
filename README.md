# RimTalk Scriban テンプレート
RimtalkがLLMに送信するプロンプトを整形するScribanテンプレートです。
ゲーム内のポーン情報・マップ情報・戦闘状態などを収集し、AIが自然なロールプレイ会話を生成しやすい形に加工します。

---

## カスタマイズ

テンプレート冒頭のユーザー設定辞書を編集することで、表示内容を変更できます。
**編集はRimTalk画面ではなく、別エディタに全文貼り付けてから行ってください。見えない改行が混じり辞書が崩れる可能性があります**

### 辞書の編集例

```scriban
{{~ keyword_tips = {
"サイトスティーラー": "サイトスティーラーは姿を消せる異形生命体です",
} ~}}
```

キーがプロンプト内に含まれていた場合、対応するValueが補足情報として末尾に追加されます。

## ペルソナ設定の書き方

各ポーンのPersonality欄に以下の書式で記述します。

### 通常テキスト
```
常に関西弁で話す
```
条件なしで常に出力されます。

### 確率指定
```
{30%}今日は気分が乗らない様子です{END}
```
指定した確率（ここでは30%）でランダムに出力されます。

### キーワードトリガー
```
{"マッファロー"}マッファローを飼育するのが趣味です{END}
```
ゲーム内の行動やイベントに指定キーワードが含まれる場合のみ出力されます。

### 戦闘時限定
```
{!}戦闘になると目つきが変わり無口になります{END}
```
戦闘状態のポーンにのみ出力されます。

## 戦闘判定ロジック

以下の条件で戦闘中と判定します。

- マップの危険度が `Low` または `High`
- かつ、直近5000tick以内に攻撃を行った or 被弾した → **「現在戦闘中」**
- かつ、直近50000tick以内に攻撃を行った or 被弾した → **「戦闘状態を継続中」**

戦闘中の場合は以下の情報が追加されます。
- 所持武器
- 戦闘時限定ペルソナ
- 戦闘時限定健康状態

## 傷病（Hediff）の表示仕様

発生からの経過時間によって表示グループが分類されます。
表示期限を設ける事で傷跡を一生気にし続けるポーンをなくします

| グループ | 経過時間の目安 |
|---|---|
| 今発生した傷病 | 約3時間以内 |
| 少し前に負った傷病 | 約半日以内 |
| 数時間前に負った傷病 | 約1日以内 |
| 数日前の傷病 | 約1週間以内 |

欠損部位（MissingBodyPart）は、頭・首・腕・脚など優先度の高い部位を1つだけ表示します。

## 注意事項

- エラーが発生した際、LLMにコードそのもの（10,000トークン程度）が送られてしまう為、トークン数を気にしなくて良いモデルで一旦テストすることを推奨します

- 使用モデルはgemmaシリーズレベルを想定しているため、ローカルモデルでは上手く動かない可能性があります
- オススメはGemini 3.1 Flash Lite（RPD 500）とGemma 4（RPD 1500）シリーズです
- `PawnsFinder.All_AliveOrDead`等を使用しているため、長時間プレイ時に負荷が増す可能性があります
- 編集時は必ず別エディタで作業し、構文エラーを防いでください
